JavaScript Browser Support

In JavaScript certain syntax features or functionalities are not supported by certain browsers. In particular modern features such as syntax features and method from ES6 and beyond do not have full browser support in older browsers such as IE. In order to combat this and allow users to have the same experience or prevent erros on older browsers we have some options such as feature detection, polyfills and transpilation.

Feature Detection

Feature detection allows us to check if a certain feature is available and supported on a browser and provide a fallback if it is not. Typically features that arent supported in a browser will return undefined so to implement feature detection we can add an if statement to check for the feature. If the feature it available then the if condition is met and we can run the code we want to do inside however if the feature is not supported and therefore returns undefined we can enter an else condition where we can provide some fallback code.

Polyfills

Polyfills allows us to add a feature to a browser thag doesnt typically support it through JavaScript. These are typically 3rd party packages that implement a feature in JavaScript code using supported methods to achieve the same type of functuonality for the unsupported broswers.

Transpilation

Transpilation allows us to write modern JavaScript code in development and then when it comes time to build our app for production the code is transpiled to support older broswers transforming these modern syntax features and methods into older JavaScript code that all browsers support e.g. const and let become var. Transpilation is done using a tool as part of your workflow that can run the transpilation on production builds. Such a tool is Babel wich is the most popualr transpilation tool for JavaScriopt code.
